웹 접근성 이란

팀 버너스 리 (웹 창시자)는 웹이란 장애에 구애없이 모든 사람들이 손쉽게 정보를 공유할 수 있는 공간 이라고 정의 하였다.

여기서, 접근성을 장애인에 국한된 문제라고 이해하면 안된다.
비록 접근성 준수가 장애인에게 가장 혜택이 많이 돌아가는 것은 사실이지만, 접근성이란 장애인뿐만 아니라 모든 사람이 정보통신 기기나 서비스를 손쉽게 활용 할 수 있도록 말하는 것이다.

기본언어 명시

웹 페이지의 기본언어를 정확히 명시해야한다. (html의 lang속성 제공 및 ISO 표준언어정의 확인)

W3C 웹 표준 준수

W3C 권고사항을 기준으로 하여 웹 표준을 준수한다.

W3C 마크업 검증사이트 https://validator.kldp.org/
의미에 맞는 마크업 및 동작

마크업이 의미에 맞게 작성되어야 한다.

시맨틱 마크업 컴퓨터가 웹 정보자원의 의미를 이해할 수 있도록 문서의 논리적인 구조를 묘사하는 것이다. 장점으로는 접근성, 코드가독성, 코드와 데이터의 재사용이 높아지는 것을 들수 있다.
시맨틱마크업 태그예제 : <header> <nav> <section> <article> <aside> <time> <figure> <address> <footer>...

모든 기능은 키보드로 사용이 가능하여야 한다. 포커스 이동 뿐만 아니라 선택을 할 경우 엔터나 스페이스바로 동작이 가능해야한다. * 알림창(alert) 또한 키보드로 사용이 가능하여야 한다.

클릭이 되어야 하는 요소는 링크(<a>나 <button>으로 마크업 되어야 하며, 여러개 중에 하나를 선택해야하는 ul의 경우 <select>나 <input type="radio">)로 마크업 해야한다. - <li>에 css / text-decoration : underline 및 cursor:pointer 으로 하면 키보드로 동작기능이 안된다.

탭 메뉴에서 탭 포커스 시 탭 내용이 선택될수 있어야한다.

이동시 포커스가 순서에 맞게 이동되지만 순서가 꼭 변경되야 할 경우 tabindex속성을 사용하여 순서를 지정한다.

tabindex는 컴퓨터의 논리적인 순서를 억지로 바꾸는 것이기 때문에 최대한 사용 하지 않는것이 좋다.

img태그

img태그가 들어가면 그 의미에 맞는 alt값을 넣어준다

단순히 '그림' , '사진' 과 같은 모호한 표현을 삼간다.

더보기 같은 경우의 이미지는 자세한 의미가 들어가야 한다. (공지사항 더보기 , 사진 더보기 ..)

의미 없는 이미지에는 빈 alt 속성이 제공 되어야 한다.

Alt값에 들어가는 내용이 많을 경우 longdesc 속성 ( longdesc=“설명경로.html” )을 이용해도 된다. * 반드시 alt속성도 제공 해야함

IR기법(Image Replacement) - 이미지 대체기법 IR기법으로 대체할수 있다.

조직도이미지는 각 항목이 대체 텍스트 뿐만아니라 각 항목 간의 관계가 표현되어야 한다.

table 태그

Table 태그 안에는 <caption>태그로 테이블의 제목값을 반드시 넣어준다.

<thead>나 <tbody>로 나눠서 의미에 맞게 사용하고 <tfoot>은 바닥글이 있을때만 사용한다. 테이블이 길 경우 리더기가 얼마나 오래 읽어야 할지 모르는 불편함을 덜기 위해 작성순서를 thead – tfoot – tboody 순으로 한다

<th> 에는 <scope> 속성을 넣거나 id 값을 지정하여 <td>에 <headers> 속성으로 연결한다.

<th>에 <abbr>속성을 이용하여 약어도 넣어줄 수 있다.

input 태그

<input>태그,<textarea>태그에는 title속성을 이용하여 역할이나 내용을 입력한다.

type="image" 인 경우 alt값과 title 값을 넣어준다.

<label>태그를 같이 쓸 경우 <input>태그에 id값을 지정하여 <label>태그의 for속성을 이용하여 연동 시켜준다.

<label> 들어갈 공간이 협소할때 <title>로 제공하여도 접근성에 위배되지 않는다.

select 태그

title 속성을 넣어 역할을 할 수 있게 한다.

<select> 태그를 이용하여 새 창을 띄울 경우(예:패밀리사이트 등) 선택시 바로 띄우지 않고 버튼 클릭시 이동하게 해야한다.

칼라구분

모든 정보는 특정한 색을 구별할 수 없는 사용자나 흑백디스플레이 사용자, 흑백인쇄물을 보는 사용자가 해당 콘텐츠를 인식 할 수 있도록 제공되어야한다.

웹 페이지가 제공하는 텍스트 콘텐츠(텍스트 및 텍스트 이미지)와 배경간의 명도 대비는 4.5:1 이상이어야 한다.

화면 확대가 가능하도록 구련한 콘텐츠(텍스트 및 텍스트이미지)의 명도대비는 3:1을 만족해야한다.

명도대비 체크툴 : Colour_Contrast_Analyser 다운받기 Colour_Contrast_Analyser

페이지 내비게이션 및 선택된 메뉴의 현재 페이지를 색상의 변화만으로 알리면 안된다. 기울임, 밑줄, 괄호, 테두리 등을 추가하여야 한다. (명도가 차이나더라도 오류임)

바른 예

회원가입 필수 항목을 색으로만 지정하면 안된다. * 표시로 구분될 수 있게 해야한다.

동영상

영상,음성 콘텐츠에 자막, 원고, 수화중 하나이상의 대체 수단을 제공하여야 한다. - 청각 장애인을 위함

텍스트만 제공하는 영상 콘텐츠는 동일한 내용의 음성을 제공하여야 한다. - 시각 장애인을 위함

자동재생

자동으로 소리가 재생되지 않아야 한다.

자동 재생음을 허용하는 경우 : 자동으로 재생되는 소리는 3초내로 멈추거나 지정된 키(예:esc키)를 누르면 재생을 멈추도록 구현한다.

사용자 요구에 의한 재생 : 콘텐츠에 포함된 멀티미디어 파일은 정지 상태로 제공하며 사용자가 요구할 경우에만 재생 할 수 있도록 제어판(멈춤,일시정지,음량조절 등)을 제공한다.

초기화면 멈춤세팅 / 지정된 키 (space바) 로 멈춤 기능 / 프레임 내 제어판 탑재

광과민성발작 예방

깜빡이거나 번쩍이는 콘텐츠로 인해 발작을 일으키지 않도록 초당 3-50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야한다. 10인치 이상의 스크린을 채용하고 있는 정보 통신 기기에서는 콘텐츠에 의한 광과민성 발작가능성을 특히 주의해야 한다.

콘텐츠 간 여백 및 구분

콘텐츠에 포함된 모든 컨트롤은 대각선 방향의 길이를 6.0 mm 이상으로 제공하는 것이 바람직하다. 링크, 사용자 입력 및 기타 컨트롤은 테두리 안쪽으로 1px 이상의 여백을 두고 이곳에서는 위치 지정 도구의 조작에 반응하지 않도록 구현하는 것이 바람직하다.

이로써, 손 떨림이 있는 사용자와 시각장애인도 컨트롤을 용이하게 찾아서 조절 할 수 있다.

이웃한 콘텐츠는 구별 될 수 있어야 한다. (테두리, 구분선, 다른 무늬, 명도or채도 달리함, 간격조절 등.)

여백이 6mm 이상의 버튼(O)
